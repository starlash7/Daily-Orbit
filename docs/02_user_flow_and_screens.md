# 02. User Flow and Screens

## 1) 정보 구조(IA)

```text
Home
 ├─ 오늘의 무료 뽑기
 ├─ 카테고리 선택(연애/금전/커리어/건강)
 └─ streak 상태

Result
 ├─ 한줄 운세
 ├─ 행운 요소(숫자/컬러/키워드)
 ├─ 공유
 └─ 심화 리딩(유료)

My
 ├─ streak 이력
 ├─ 최근 결과(차기)
 └─ 구매 이력(차기)
```

## 2) 핵심 사용자 플로우

### 플로우 A: 무료 체험

1. 홈 진입
2. 카테고리 선택
3. 무료 뽑기 클릭
4. 결과 확인
5. 공유 또는 종료

### 플로우 B: 유료 전환

1. 결과 화면 진입
2. 심화 리딩 CTA 클릭
3. 결제 진행(USDC)
4. 성공 시 상세 해석 노출
5. 실패 시 재시도/취소 안내

### 플로우 C: 재방문 루프

1. 다음날 재진입
2. 무료 뽑기 실행
3. streak 갱신
4. milestone(3/7/14/30일) 피드백 제공

## 3) 화면 요구사항

## Screen 1: Home

- 목적: 즉시 뽑기 유도
- 주요 컴포넌트
  - 오늘의 CTA 버튼
  - 카테고리 탭 4개
  - streak 위젯
  - 캐릭터 히어로 일러스트
- 핵심 CTA
  - `오늘의 운세 뽑기`
- 상태
  - 빈 상태: 첫 방문 설명 카드
  - 에러: 네트워크 오류 재시도 버튼

## Screen 2: Result

- 목적: 만족감 + 공유 유도
- 주요 컴포넌트
  - 결과 카드(문구 + 행운 요소)
  - 캐릭터 반응 표정
  - 공유 버튼
  - 심화 리딩 잠금 영역
- 핵심 CTA
  - `결과 공유하기`
  - `심화 해석 열기`
- 상태
  - 무료 소진 상태: "내일 무료 뽑기 가능" 안내

## Screen 3: Deep Reading (Paid)

- 목적: 결제 가치 명확화
- 주요 컴포넌트
  - 상세 해석 4블록(기회/주의/관계/행동)
  - 결제 금액/혜택 안내
  - 결제 버튼
- 결제 실패 상태
  - 사용자가 이해 가능한 실패 이유 표시
  - 재시도/취소/고객지원 링크 제공

## Screen 4: My

- 목적: 재방문 동기 강화
- 주요 컴포넌트
  - 현재 streak/최고 streak
  - 최근 7일 요약(차기)
  - 배지/마일스톤

## 4) 카피 원칙

- 짧고 명확하게(모바일 1~2행)
- 불안 조장 금지
- 행동 가능한 제안 포함

예시:

- 좋은 예: "오늘은 속도를 줄이면 실수가 줄어들어요."
- 나쁜 예: "오늘 큰 불행이 닥칩니다."

## 5) 예외 시나리오

| 시나리오 | 처리 원칙 |
|---|---|
| 네트워크 오류 | 재시도 버튼 + 직전 상태 보존 |
| 결제 승인 실패 | 원인 메시지 + 재시도 경로 |
| 무료 뽑기 중복 요청 | 서버에서 idempotency 처리 |
| 데이터 로드 지연 | skeleton/로딩 인디케이터 |

## 6) 접근성/모바일 기준

- 터치 타겟 최소 44px
- 결과 텍스트 명도 대비 확보
- 버튼 문구는 동사형으로 명확화
- 세로 화면 우선 설계(모바일 first)
