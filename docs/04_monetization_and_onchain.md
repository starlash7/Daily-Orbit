# 04. Monetization and Onchain Strategy

## 1) 수익화 원칙

- 코어 경험은 무료로 제공한다
- 결제는 감정 몰입 이후에 노출한다
- 결제 가치는 "더 깊은 해석"으로 명확히 설명한다

## 2) 무료/유료 경계

| 구분 | 제공 내용 |
|---|---|
| 무료 | 일일 1회 운세, 기본 결과 카드, streak 적립 |
| 유료 | 심화 리딩(기회/주의/관계/행동), 추가 뽑기(옵션) |

## 3) 가격 가설 (초기)

- 심화 리딩 1회: 0.3 / 0.5 / 0.7 USDC A/B 테스트
- 추가 뽑기 1회: 0.1 / 0.2 / 0.3 USDC A/B 테스트
- 주간 패스(v1.1): 1.99 USDC

## 4) Paywall 노출 지점

1. 결과 화면 하단: "오늘의 상세 해석 열기"
2. 무료 소진 후 재뽑기 시점
3. streak milestone 달성 시 특별 해석 제안

노출 원칙:

- 첫 결과를 충분히 보여준 뒤 유료 전환 제안
- 한 화면에 결제 CTA는 1개만 유지

## 5) 온체인/결제 UX

### 사용자 관점

- 무료 기능은 코인 없이도 사용 가능
- 유료 결제 시 USDC로 명확히 표시
- 결제 성공 후 즉시 컨텐츠 잠금 해제

### 시스템 관점

- 결제 자산: Base 네트워크 USDC
- 트랜잭션 상태: `pending`, `confirmed`, `failed` 관리
- 실패 처리: 재시도 버튼 + 취소 복귀 경로

## 6) 가스 UX 전략

- 원칙: 사용자가 "가스"를 인식하지 않도록 단순화
- 가능하면 sponsored tx/paymaster로 체감 비용 완화
- 가스가 필요한 경우 사전 안내 문구 노출

예시 문구:

- "결제가 완료되면 상세 운세가 바로 열려요."
- "네트워크 상태에 따라 승인에 몇 초 걸릴 수 있어요."

## 7) 결제 실패/예외 처리

| 실패 유형 | 사용자 메시지 | 액션 |
|---|---|---|
| 사용자 취소 | 결제가 취소되었어요 | 다시 시도 |
| 잔액 부족 | USDC 잔액이 부족해요 | 충전 안내 |
| 네트워크 지연 | 확인 중이에요 | 상태 새로고침 |
| 처리 실패 | 결제 확인에 실패했어요 | 문의/재시도 |

## 8) 수익화 실험 계획

- 가격 실험: 2~3개 가격 포인트
- 카피 실험: 가치 강조 문구 A/B
- 타이밍 실험: 결과 직후 vs streak 달성 직후

## 9) 운영 리스크

1. 결제 UX가 복잡하면 전환률 급락
2. 가격이 높으면 체험-결제 간 단절 발생
3. 오류 메시지가 불명확하면 신뢰도 하락

## 10) 가정

- 초기 결제량은 소액 다건 구조를 목표로 함
- 법/정책 검토는 출시 전 별도 점검 수행
- 환불 정책은 간단한 FAQ로 시작 후 고도화
